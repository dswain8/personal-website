# DACI Generator — Create a Decision Document

Create a DACI decision document for a cross-functional initiative or product decision.

Input: $ARGUMENTS — describe the decision, initiative, or paste relevant context/Slack threads.

## Step 1 — Extract the key decision

Identify:
- What is the decision that needs to be made?
- What are the realistic options?
- Who has a stake in this?

## Step 2 — Write the DACI doc

```
# DACI: [Decision / Initiative Name]

**Date:** [today's date]
**Status:** Draft
**DRI:** [Your Name]

---

## Context

[What is this about? Why does a decision need to be made now? What happens if we don't decide?]

## Decision Options

| Option | Description | Pros | Cons |
|--------|-------------|------|------|
| Option 1 | | | |
| Option 2 | | | |

## Roles

| Role | Person(s) | What they do |
|------|-----------|--------------|
| **Driver** | [Your Name] | Owns the process, drives to a decision |
| **Approver** | [Name] | Final decision authority — their "yes" closes it |
| **Consulted** | [Names] | Must be heard before decision is made |
| **Informed** | [Names] | Notified after decision, no input required |

## Recommended Decision

[Which option, and why. Be direct.]

## Open Questions

| Question | Owner | Due date |
|----------|-------|----------|
| | | |

## Decision Log

| Date | Decision made | By |
|------|---------------|----|
| | | |
```

## Step 3 — Save and confirm

Save to your meetings or decisions folder with today's date.
Present for review. Do NOT share with anyone without explicit approval.
